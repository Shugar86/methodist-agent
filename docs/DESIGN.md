# Methodist Agent — Design & UX Architecture

> Высокоуровневое описание пользовательского опыта, интерфейсов и пользовательских путей Methodist Agent.

## Философия дизайна

Methodist Agent строится вокруг привычного рабочего места методиста — Windows + Microsoft Office. Интерфейс минимален, не отвлекает от работы и появляется только тогда, когда нужен.

**Ключевые принципы:**
1. **Работай там, где пользователь.** System Tray и контекстное меню Explorer — первичные точки входа. CLI — для продвинутых сценариев.
2. **Минимум кликов до результата.** Создание частых документов — не более 2–3 действий.
3. **Прозрачность действий.** Пользователь всегда видит, что агент собирается сделать, и может подтвердить или отменить.
4. **Graceful degradation.** Если COM/Office недоступен, агент переключается на native-режим и продолжает работу.

## Пользовательские пути

### 1. Быстрое создание документа из System Tray

```
Клик по иконке в трее
  → «Создать документ»
    → Выбор типа (Рабочая программа / Ведомость / Презентация / Отчёт)
      → Диалог с минимумом полей (дисциплина, часы/группа)
        → Подтверждение
          → Создание файла
            → Уведомление + открытие папки
```

### 2. Создание через chat

```
Chat
  → Пользователь пишет запрос
    → Orchestrator анализирует и предлагает план
      → Approval gate
        → Выполнение
          → Результат
```

### 3. Адаптация документа

```
Правый клик по DOCX в Explorer
  → «Адаптировать с помощью Методист-Агента»
    → Выбор целевого шаблона/ФГОС
      → Создание новой версии
        → Уведомление
```

### 4. Batch-обработка

```
Chat/CLI
  → "Обнови все рабочие программы в папке X под новый шаблон"
    → План с перечнем файлов
      → Approval
        → Batch-обработка
          → Отчёт о проделанной работе
```

## Компоненты интерфейса

### System Tray App

- **Иконка:** синий круг с белой буквой «М» (уже реализовано).
- **Меню по левому клику:**
  - 📝 Создать документ → подменю
  - 🔍 Поиск методических материалов
  - 📂 Открыть папку с документами
  - ⚙️ Настройки
  - ❌ Выход
- **Уведомления:** краткие, с конкретикой. Пример: `✅ Рабочая программа готова: Базы_данных.docx`.

### Диалоги создания документа

Минималистичные tkinter-диалоги с полями:
- Рабочая программа: дисциплина, часы, направление/группа.
- Ведомость: дисциплина, группа, семестр.
- Презентация: тема, количество слайдов (опционально).
- Отчёт: период, тип отчёта.

Кнопки: **Создать** (primary), **Отмена**.

### Chat UI

В текущей версии — CLI chat. В перспективе — встроенное окно с историей.

- **История сообщений** с разделением user/assistant.
- **План действий** перед выполнением.
- **Кнопки approval:** Подтвердить / Отменить.
- **Индикатор прогресса** при выполнении задач.

### CLI

- Используется для автоматизации, скриптов и отладки.
- Команды: `init`, `create`, `adapt`, `search`, `pdf`, `skills`, `config`, `tray`.
- Все сообщения CLI должны соответствовать `UX_GUIDE.md`.

### Onboarding

1. **Приветствие** — `onboarding_welcome()`.
2. **Проверка окружения** — `run_environment_check()` via `init --check` or after normal `init`.
3. **Создание первого документа** — `onboarding_first_step()`.
4. **Повторная проверка** — command `methodist-agent init --check` anytime.

## Empty States

| Состояние | Сообщение |
|-----------|-----------|
| Нет сессий | `Пока нет сохранённых диалогов. Начните с команды «создай рабочую программу».` |
| Нет skills | `Skills не загружены. Запустите инициализацию.` |
| Нет шаблонов | `Шаблоны не найдены. Проверьте папку templates/ или запустите init.` |
| Поиск не дал результатов | `По запросу ничего не найдено. Попробуйте уточнить тему или тип материала.` |

## Error States

Все ошибки должны:
1. Объяснять, что случилось, простым языком.
2. Предлагать следующий шаг.
3. Не содержать traceback в UI.

Реализация проверки окружения — `src/core/environment_check.py`; пользовательские тексты ошибок и onboarding-сообщений — `src/core/ui_text.py`.

| Ошибка | Сообщение пользователю |
|--------|------------------------|
| Нет API-ключа | `Не удалось подключиться к языковой модели. Проверьте API-ключ в настройках или выберите локальную модель Ollama.` |
| Нет Office/COM | `Word/Excel не найдены. Переключаюсь на встроенный режим — документ всё равно будет готов.` |
| Файл не найден | `Не удалось найти файл: путь. Проверьте, что путь указан верно.` |
| Ошибка записи | `Не удалось сохранить файл. Проверьте права на запись в папку.` |

## Контекстное меню Explorer (план)

Регистрация в Windows Registry:

- `Создать с помощью Методист-Агента` — для папок.
- `Адаптировать с помощью Методист-Агента` — для DOCX/XLSX/PPTX/PDF.
