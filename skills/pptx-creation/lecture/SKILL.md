# Создание лекционной презентации

## Контекст
Лекционная презентация — визуальный методический материал, используемый преподавателем при проведении лекционных занятий. Должна соответствовать требованиям к оформлению учебных материалов вуза и принципам педагогической эргономики.

## Шаги

1. **Уточнить параметры лекции**
   - Тема лекции, дисциплина, курс
   - Количество слайдов (рекомендуется 10–20 на 2 академических часа)
   - Уровень сложности: вводная, основная, углублённая
   - Целевая аудитория: бакалавриат/магистратура
   - Наличие методических указаний или РПД с тематическим планом

2. **Разработать структуру презентации**
   - **Слайд 1. Титульный**: тема лекции, дисциплина, ФИО преподавателя, ОО, дата
   - **Слайд 2. Цели и задачи**: 2–3 цели лекции, что студент должен знать/уметь после изучения
   - **Слайд 3. План лекции**: перечень основных вопросов (3–5 пунктов)
   - **Основные слайды (4–N)**: по одному ключевому тезису на слайд
   - **Слайд «Пример/Кейс»**: иллюстрация теории практическим примером
   - **Слайд «Проверка знаний»**: 2–3 контрольных вопроса или тестовое задание
   - **Последний слайд**: основные выводы, литература для самостоятельной работы

3. **Оформить слайды**
   - **Шаблон**: использовать корпоративный шаблон вуза (цвета, логотип, шрифты)
   - **Текст**: не более 6 строк на слайд, не более 6 слов в строке
   - **Заголовки**: краткие, информативные (не «Введение», а «Определение базы данных»)
   - **Шрифты**: заголовок 28–32 пт, основной текст 20–24 пт
   - **Цвета**: тёмный текст на светлом фоне, акцентный цвет для ключевых терминов
   - **Медиа**: схемы, диаграммы, фотографии, короткие видео (до 2 мин)

4. **Наполнить содержанием**
   - Каждый слайд должен раскрывать один тезис или понятие
   - Использовать структурированные списки вместо сплошного текста
   - Добавить схемы, блок-схемы, UML-диаграммы, графики (где уместно)
   - Код/формулы — моноширинным шрифтом, с подсветкой синтаксиса
   - Примеры — из реальной практики, актуальные для отрасли
   - Источники — научные статьи, учебники ФГОС 3++, официальная документация

5. **Проверить и сохранить**
   - Проверить читаемость с задних рядов аудитории (текст крупный, контрастный)
   - Убедиться, что анимации не отвлекают от содержания (применять умеренно)
   - Проверить все гиперссылки и встроенные объекты
   - Сохранить как: `Лекция_<Тема>_<Дисциплина>.pptx`
   - Дополнительно: экспорт в `.pdf` для публикации в LMS

## Правила

- На слайде должен быть один центральный тезис — не пытаться уместить всю лекцию на одном слайде
- Не использовать анимации «распыление», «подпрыгивание», «вращение» — только простые появления
- Не размещать текст «просто для заполнения пространства» — каждое слово должно нести смысл
- Изображения должны быть в высоком разрешении (не менее 150 dpi при печати)
- Цитаты и данные должны содержать ссылку на источник
- Не использовать скриншоты с водяными знаками стоковых фотобанков
- Соотношение сторон: 16:9 (современный стандарт), 4:3 только по требованию вуза
- Общий объём файла не должен превышать 50 МБ (для загрузки в LMS)

## Пример

**Запрос пользователя:** «Создай презентацию по лекции „Нормализация баз данных“ для 3 курса, дисциплина „Базы данных“»

**Действия агента:**
1. Запрашивает: количество слайдов (15), наличие шаблона вуза
2. Структура:
   - Слайд 1: Титул
   - Слайд 2: Цели (понять цели нормализации, знать нормальные формы до 3НФ)
   - Слайд 3: План (1. Проблемы избыточности, 2. 1НФ, 3. 2НФ, 4. 3НФ, 5. Практический пример)
   - Слайды 4–5: Проблемы (аномалии вставки, обновления, удаления) со схемой
   - Слайды 6–8: 1НФ, 2НФ, 3НФ — определения + примеры таблиц
   - Слайд 9: Пошаговый пример нормализации таблицы «Заказы»
   - Слайд 10: Проверка знаний (3 вопроса)
   - Слайд 11: Выводы + литература (Дейт К.Дж. «Введение в системы баз данных»)
3. Оформляет в корпоративном шаблоне, шрифт Calibri 24–32 пт
4. Добавляет схемы: исходная таблица → 1НФ → 2НФ → 3НФ
5. Сохраняет: `Лекция_Нормализация_БД_3курс.pptx`

## Проверка качества

- [ ] Структура соответствует тематическому плану РПД
- [ ] Количество слайдов адекватно времени лекции (10–20 на 2 часа)
- [ ] На каждом слайде не более 6 строк текста
- [ ] Шрифт крупный и читаемый (минимум 20 пт)
- [ ] Использован корпоративный шаблон вуза
- [ ] Есть схемы/диаграммы для сложных понятий
- [ ] Примеры актуальны и практичны
- [ ] Анимации минимальны и не отвлекают
- [ ] Последний слайд содержит выводы и литературу
- [ ] Файл сохранён с корректным именем
