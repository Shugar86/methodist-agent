# Генерация аналитического отчёта

## Контекст
Аналитический отчёт по данным образовательного процесса позволяет методисту выявлять тренды, проблемные зоны, оценивать эффективность учебных программ и принимать обоснованные решения по их улучшению. Источники данных: ведомости успеваемости, опросы студентов, статистика посещаемости, данные LMS.

## Шаги

1. **Определить цель и источники данных**
   - Запросить у пользователя:
     - Цель анализа: успеваемость, посещаемость, востребованность дисциплин, сравнение групп/потоков
     - Период: семестр, учебный год, несколько лет
     - Источники данных: файлы `.xlsx`, `.csv`, выгрузки из 1С/Мoodle, результаты опросов
     - Уровень детализации: по группам, по дисциплинам, по преподавателям, по курсам
   - Собрать все необходимые файлы данных

2. **Загрузить и очистить данные**
   - Загрузить данные из предоставленных файлов
   - Проверить целостность:
     - Отсутствующие значения (NULL, пустые ячейки)
     - Дубликаты записей
     - Некорректные форматы (даты, числа, текстовые оценки)
     - Аномалии (оценка «6» в 5-балльной системе, отрицательные часы)
   - Очистить и стандартизировать:
     - Привести ФИО к единому формату (Фамилия И.О.)
     - Унифицировать названия дисциплин (убрать лишние пробелы, разный регистр)
     - Преобразовать оценки в числовой формат
   - Создать единый датасет для анализа

3. **Выполнить анализ**
   - **Описательная статистика**:
     - Средний балл, медиана, мода, стандартное отклонение
     - Распределение оценок (отлично/хорошо/удовл./неуд.)
     - Процент успеваемости (сдано / всего)
     - Процент качества знаний (отлично + хорошо / всего)
   - **Сравнительный анализ**:
     - Динамика по семестрам/годам
     - Сравнение групп (параллельных, разных форм обучения)
     - Сравнение дисциплин (базовых и вариативных)
   - **Корреляционный анализ** (при наличии данных):
     - Связь посещаемости и успеваемости
     - Связь нагрузки преподавателя и результатов студентов
     - Связь СРС и итоговых оценок
   - **Выявление аномалий**:
     - Дисциплины с аномально низкой успеваемостью (< 50%)
     - Группы с резким падением/ростом показателей
     - Преподаватели с систематически отличающимися результатами

4. **Визуализировать результаты**
   - **Диаграммы**:
     - Столбчатые: распределение оценок по дисциплинам/группам
     - Линейные: динамика успеваемости по периодам
     - Круговые: доли категорий (отлично/хорошо/удовл./неуд.)
     - Тепловые карты: успеваемость по группам × дисциплины
   - **Таблицы**:
     - Сводная таблица по дисциплинам (средний балл, % успеваемости, % качества)
     - Рейтинг групп/преподавателей
   - **Требования к оформлению**:
     - Заголовки диаграмм информативные
     - Подписи осей, легенда
     - Цветовая палитра: зелёный (хорошо), жёлтый (средне), красный (проблема)
     - Разрешение: минимум 150 dpi для печати

5. **Сформировать отчёт**
   - **Структура отчёта**:
     - Титульный лист
     - Введение: цель, период, источники данных
     - Раздел 1. Общая характеристика выборки (количество студентов, дисциплин, групп)
     - Раздел 2. Основные результаты (с диаграммами и таблицами)
     - Раздел 3. Выявленные проблемы и тренды
     - Раздел 4. Рекомендации (конкретные, адресованные кафедре/деканату)
     - Приложения: исходные данные, детальные таблицы
   - **Требования к тексту**:
     - Каждое утверждение подкреплено цифрой или диаграммой
     - Рекомендации измеримы («увеличить часы практики на 20%» вместо «добавить больше практики»)
     - Сравнения с предыдущими периодами (если данные доступны)
   - Сохранить как: `Аналитический_отчёт_<Тема>_<Период>.docx` + приложения `.xlsx`

## Правила

- Не делать выводов на основе неполных или некорректных данных — сначала указать на пробелы
- Не публиковать персональные данные студентов (ФИО, номера зачёток) в открытых отчётах — использовать обезличенные ID
- Сравнение преподавателей проводить только внутри одной дисциплины/группы, а не между разными предметами
- Аномалии требуют проверки: низкая успеваемость может быть связана с объективной сложностью дисциплины, а не с качеством преподавания
- Все расчёты должны быть воспроизводимы — сохранять формулы и скрипты
- Для выборок менее 10 человек статистические выводы ограничивать («тенденция» вместо «закономерность»)
- Соблюдать требования к защите персональных данных (152-ФЗ)

## Пример

**Запрос пользователя:** «Сделай аналитику успеваемости групп ПИ-301 и ПИ-302 за осенний семестр 2025»

**Действия агента:**
1. Получает файлы: `Ведомости_осень2025.xlsx` (5 дисциплин, 2 группы, 50 студентов)
2. Очищает данные:
   - Находит 2 пропущенных значения (н/я → помечает как отсутствие)
   - Унифицирует название дисциплины «БД» → «Базы данных»
3. Анализирует:
   - Средний балл ПИ-301: 4.1, ПИ-302: 3.7
   - % качества (отл.+хор.): ПИ-301: 72%, ПИ-302: 58%
   - Проблемная дисциплина: «Математическая статистика» — средний балл 3.2, 15% неуд.
   - Корреляция посещаемости и оценки: r = 0.68 (умеренная положительная)
4. Визуализирует:
   - Столбчатая диаграмма: сравнение групп по дисциплинам
   - Линейный график: динамика среднего балла по неделям
   - Тепловая карта: оценки студентов × дисциплины
5. Формирует отчёт (6 страниц):
   - Рекомендация 1: для ПИ-302 организовать консультации по мат. статистике
   - Рекомендация 2: рассмотреть увеличение часов практики по мат. статистике на 20%
6. Сохраняет:
   - `Аналитический_отчёт_успеваемость_ПИ_осень2025.docx`
   - `Данные_и_диаграммы_ПИ_осень2025.xlsx`

## Проверка качества

- [ ] Данные загружены полностью и очищены от ошибок
- [ ] Описательная статистика рассчитана корректно
- [ ] Диаграммы информативны и корректно подписаны
- [ ] Все выводы подкреплены данными
- [ ] Персональные данные обезличены (в открытых отчётах)
- [ ] Рекомендации конкретны и измеримы
- [ ] Отчёт содержит сравнение с предыдущими периодами (если возможно)
- [ ] Аномалии проверены на объективные причины
- [ ] Формулы/скрипты сохранены для воспроизводимости
- [ ] Файлы сохранены с корректными именами
