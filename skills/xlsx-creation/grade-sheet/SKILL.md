# Создание ведомости успеваемости

## Контекст
Ведомость успеваемости — официальный документ, фиксирующий результаты промежуточной аттестации студентов по дисциплине. Используется для внесения оценок в деканат, электронную систему вуза (1С:Университет, Moodle и др.) и формирования приложения к диплому.

## Шаги

1. **Запросить параметры ведомости**
   - Дисциплина, направление подготовки, курс, группа(ы)
   - Семестр, учебный год
   - Вид аттестации: зачёт / зачёт с оценкой / дифференцированный зачёт / экзамен / курсовая работа
   - Форма контроля: текущий (по модулям) или итоговый
   - Количество студентов или список групп

2. **Сформировать структуру таблицы**
   - **Столбцы обязательные**: № п/п, ФИО студента, номер зачётной книжки
   - **Столбцы оценок** (зависят от вида аттестации):
     - Зачёт: «зачтено» / «не зачтено»
     - Зачёт с оценкой / экзамен: отметка (5, 4, 3, 2) + зачёт/незачёт
     - Дифференцированный зачёт: баллы (0–100) + оценка (5/4/3/2)
     - По модулям: столбцы для каждого модуля (М1, М2, М3…) + итог
   - **Дополнительные столбцы**: пропуски (уваж./неуваж.), дата, подпись преподавателя

3. **Заполнить список студентов**
   - Запросить список у пользователя или загрузить из файла группы (`.xlsx`, `.csv`, `.txt`)
   - Проверить: ФИО в именительном падеже, корректность номеров зачётных книжек
   - Отсортировать по алфавиту (фамилия → имя → отчество)
   - Пронумеровать строки непрерывно

4. **Настроить систему оценивания**
   - Для дифференцированного зачёта/экзамена с баллами:
     - Определить шкалу (например, ФГОС 3++ рекомендует: 86–100 = отлично, 71–85 = хорошо, 51–70 = удовлетворительно, 0–50 = неудовлетворительно)
     - Или использовать шкалу вуза
   - Для модульного контроля: распределить баллы по модулям (сумма = 100)
   - Добавить формулы автоматического пересчёта баллов в оценки (если ведомость в Excel)

5. **Оформить и сохранить**
   - Шапка ведомости: наименование ОО, факультет, кафедра, дисциплина, группа, семестр, вид аттестации, дата
   - Границы таблицы: чёткие линии, заголовки жирным
   - Фиксация шапки при прокрутке (для Excel)
   - Защита формул от случайного редактирования
   - Сохранить как: `Ведомость_<Дисциплина>_<Группа>_<Семестр>.xlsx`
   - Дублировать в `.pdf` для подписи (при необходимости)

## Правила

- Количество студентов в ведомости должно точно соответствовать списку группы на момент аттестации
- ФИО студентов — строго в именительном падеже (не «Иванова», а «Иванова А.А.»)
- Номера зачётных книжек проверять на формат (буквы, цифры, год)
- Оценки «2» и «неудовлетворительно» выделять цветом (красный) для визуального контроля
- Пустые ячейки оценок запрещены — если студент не явился, ставить «н/я» или «2» согласно регламенту вуза
- Для зачёта с оценкой нельзя ставить «зачтено» без цифровой оценки
- Все формулы пересчёта должны быть проверены на корректность (тестовые данные)

## Пример

**Запрос пользователя:** «Создай ведомость по дисциплине „Базы данных“, группа ПИ-301, 5 семестр, экзамен, 25 студентов»

**Действия агента:**
1. Запрашивает: учебный год (2025/2026), систему оценивания (5-балльная), список студентов
2. Получает список из 25 студентов, проверяет ФИО и зачётные книжки
3. Создаёт таблицу Excel:
   - Столбцы: №, ФИО, № зач. книжки, Модуль 1 (30 баллов), Модуль 2 (30 баллов), Модуль 3 (30 баллов), Экзамен (10 баллов), Сумма, Оценка
   - Формула оценки: `=ЕСЛИ(H2>=86;5;ЕСЛИ(H2>=71;4;ЕСЛИ(H2>=51;3;2)))`
4. Заливает ячейки с оценкой «2» красным цветом
5. Блокирует формулы, оставляет редактирование только ячеек ввода баллов
6. Сохраняет: `Ведомость_Базы_данных_ПИ-301_5сем.xlsx`

## Проверка качества

- [ ] Списки студентов актуальны и отсортированы по алфавиту
- [ ] Номера зачётных книжек корректны и уникальны
- [ ] Шкала оценивания соответствует регламенту вуза/ФГОС 3++
- [ ] Формулы пересчёта работают корректно (проверено на тестовых данных)
- [ ] Сумма баллов по модулям равна 100 (или итоговому максимуму)
- [ ] Оценки «2» / «неудовлетворительно» визуально выделены
- [ ] Нет пустых ячеек в столбцах оценок
- [ ] Шапка содержит все обязательные реквизиты
- [ ] Файл сохранён с корректным именем
