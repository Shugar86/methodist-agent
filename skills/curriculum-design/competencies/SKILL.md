# Проектирование компетенций

## Контекст
Компетенции — ключевой элемент образовательной программы по ФГОС 3++. Они определяют, что выпускник должен знать, уметь и применять по окончании обучения. Проектирование компетенций включает формулировку, структурирование, распределение по дисциплинам и разработку индикаторов достижения.

## Шаги

1. **Определить контекст проектирования**
   - Запросить у пользователя:
     - Уровень образования: бакалавриат, магистратура, специалитет, СПО
     - Направление подготовки (код и название по ФГОС 3++)
     - Профиль (если применимо)
     - Цель: разработка новой ОП, обновление существующей, проектирование дисциплины
     - Отраслевые стандарты или требования работодателей (при наличии)

2. **Изучить требования ФГОС 3++**
   - Загрузить актуальный ФГОС для направления подготовки
   - Выделить обязательные (универсальные) компетенции:
     - УК-1…УК-8 (универсальные компетенции) — общие для всех направлений
   - Выделить общепрофессиональные компетенции (ОПК) — специфичные для профессиональной области
   - Выделить профессиональные компетенции (ПК) — специфичные для направления/профиля
   - Проверить: в ФГОС 3++ компетенции могут быть дополнены вузом, но не могут быть исключены обязательные

3. **Разработать/уточнить компетенции**
   - **Для новой ОП или профиля**:
     - Провести анализ требований рынка труда: вакансии, профстандарты, отраслевые исследования
     - Сформулировать ПК по принципу: «выпускник должен уметь … в контексте … с использованием …»
     - Каждая ПК должна включать: знания, умения, владения (практические навыки)
     - Пример хорошей формулировки ПК:
       > «ПК-5: способен разрабатывать программное обеспечение с использованием современных языков программирования, фреймворков и систем контроля версий в соответствии с требованиями технического задания»
   - **Для существующей ОП**:
     - Проверить соответствие формулировок ФГОС 3++
     - Уточнить индикаторы достижения (измеримые критерии)
     - Дополнить компетенциями, отражающими новые тренды (цифровизация, ИИ, soft skills)

4. **Распределить компетенции по дисциплинам**
   - Создать матрицу соответствия: компетенции × дисциплины
   - **Принципы распределения**:
     - Каждая компетенция должна формироваться не менее чем в 2–3 дисциплинах (нарастающий эффект)
     - Базовые дисциплины формируют фундамент (УК, ОПК)
     - Профильные дисциплины формируют специализацию (ПК)
     - Практики и НИР закрепляют компетенции в реальной деятельности
   - **Для каждой ячейки матрицы указать**:
     - Уровень формирования: введение (I), закрепление (Z), совершенствование (S)
     - Форма контроля: тест, проект, защита, экзамен
   - Проверить: нет «висячих» компетенций (не привязанных к дисциплинам)

5. **Разработать индикаторы и оценочные средства**
   - Для каждой компетенции сформулировать 2–4 индикатора достижения
   - **Индикатор должен быть измеримым**:
     - Плохо: «знает основы программирования»
     - Хорошо: «разрабатывает консольное приложение на Python с использованием ООП, которое проходит 10 тестовых сценариев»
   - Привязать индикаторы к конкретным заданиям в дисциплинах:
     - Лабораторная работа №3 → ПК-5 (уровень Z)
     - Курсовой проект → ПК-5 (уровень S)
   - Внести индикаторы в ФОС соответствующих дисциплин

6. **Оформить и согласовать**
   - Создать документ «Компетенции выпускника»:
     - Титульный лист
     - Перечень УК, ОПК, ПК с формулировками
     - Матрица распределения по дисциплинам
     - Индикаторы достижения
   - Сохранить как: `Компетенции_<Направление>_<Профиль>_<Год>.docx`
   - Дополнительно: экспорт матрицы в `.xlsx` для удобства редактирования

## Правила

- Не придумывать компетенции «с нуля» для существующих направлений — использовать ФГОС 3++ как основу
- Формулировки компетенций должны быть глаголами действия (способен, умеет, владеет), а не существительными («знание программирования»)
- Каждая компетенция должна быть проверяема — если невозможно измерить, переформулировать
- УК (универсальные компетенции) нельзя исключать или существенно изменять — они установлены ФГОС 3++
- ПК должны отражать специфику профиля: для «Программной инженерии» — разработка ПО, для «Информационной безопасности» — защита информации
- Матрица распределения не должна быть «равномерной» — ключевые дисциплины формируют ключевые компетенции
- Индикаторы должны быть согласованы с преподавателями, ведущими соответствующие дисциплины

## Пример

**Запрос пользователя:** «Спроектируй компетенции для нового профиля „Разработка мобильных приложений“ направления 09.03.04»

**Действия агента:**
1. Загружает ФГОС 3++ для 09.03.04, извлекает УК и ОПК
2. Анализирует рынок: 15 вакансий «Android/iOS разработчик», профстандарт 06-015
3. Формулирует ПК:
   - ПК-М1: способен проектировать архитектуру мобильного приложения с учётом ограничений мобильных устройств
   - ПК-М2: способен разрабатывать нативные приложения для Android (Kotlin) и iOS (Swift)
   - ПК-М3: способен интегрировать мобильные приложения с REST API и внешними сервисами
   - ПК-М4: способен обеспечивать безопасность данных в мобильных приложениях
4. Распределяет по дисциплинам:
   - «Программирование» → ПК-М1 (I), ПК-М2 (I)
   - «Разработка мобильных приложений» → ПК-М1 (Z), ПК-М2 (Z), ПК-М3 (I)
   - «Проектный практикум» → ПК-М1 (S), ПК-М2 (S), ПК-М3 (Z), ПК-М4 (Z)
   - «Преддипломная практика» → все ПК-М (S)
5. Разрабатывает индикаторы:
   - ПК-М2, индикатор 1: «Разрабатывает Android-приложение с 3+ экранами, навигацией и подключением к API, проходящее code review»
6. Сохраняет:
   - `Компетенции_09.03.04_Мобильные_приложения_2026.docx`
   - `Матрица_09.03.04_Мобильные_приложения_2026.xlsx`

## Проверка качества

- [ ] Использован актуальный ФГОС 3++ для направления
- [ ] Все обязательные УК включены без изменений
- [ ] ОПК соответствуют профессиональной области
- [ ] ПК отражают специфику профиля и требования рынка труда
- [ ] Формулировки компетенций содержат глаголы действия
- [ ] Каждая компетенция проверяема (есть индикаторы)
- [ ] Матрица распределения покрывает все компетенции (нет «висячих»)
- [ ] Каждая компетенция формируется в ≥ 2 дисциплинах
- [ ] Индикаторы измеримы и привязаны к конкретным заданиям
- [ ] Документы сохранены в `.docx` и `.xlsx`
